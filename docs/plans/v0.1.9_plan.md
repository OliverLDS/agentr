# agentr 0.1.9 Plan

`0.1.9` should introduce public dedicated proposal and state classes on top of the `0.1.8` lifecycle and persistence surface.

## Release Goal

Move workflow proposal handling from plain persisted records into explicit public R6 classes that match the package's existing object model.

This release should:

- add a public `WorkflowProposal` class
- add a public `WorkflowProposalState` class
- integrate `Scaffolder` with the new state class while preserving current top-level methods
- keep proposal persistence helpers working across class-based and record-based inputs

This release should not:

- redesign workflow specs
- remove the existing `Scaffolder` facade
- break current proposal review flows

## Public Class Scope

### `WorkflowProposal`

Public responsibilities:

- hold one workflow proposal and its lifecycle metadata
- validate proposal shape
- transition proposal status
- attach discussion rounds
- summarize itself
- export graph data
- save itself

### `WorkflowProposalState`

Public responsibilities:

- hold the current approved workflow
- store proposal history
- add, retrieve, list, and approve proposals
- expose latest and active proposals
- keep approval and supersession logic in one place

## Integration Rules

- `Scaffolder` keeps its current public methods
- `Scaffolder` gains a public `workflow_state` field
- `Scaffolder$workflow` remains available for backward compatibility and mirrors `workflow_state$approved_workflow`
- proposal-returning methods should return `WorkflowProposal` objects publicly
- persistence helpers should accept both `WorkflowProposal` objects and old proposal records where practical

## Tests

Add or update tests for:

- `WorkflowProposal$new()` and lifecycle methods
- `WorkflowProposalState$new()` and approval flow
- `load_workflow_proposal()` returning a public class object
- `Scaffolder` proposal methods returning and using public class objects
- backward-compatible graph and validation helpers

## Definition of Done

`0.1.9` is done when:

- the new public classes are exported and documented
- `Scaffolder` uses `WorkflowProposalState` internally
- proposal persistence still works
- isolated tests pass
- version/docs align on `0.1.9`
