# Table 2. Subsystems In agentr And Their Intended Roles

Caption: Subsystems in `agentr` and their intended roles.

| Subsystem | Full name | Conceptual role | Typical need in a task |
|---|---|---|---|
| `RWM` | Reflective Working Memory | Maintains reflective state across steps, including memory, context integration, and optional affective modulation. | Needed when the task requires persistent context, memory of prior decisions, or reflective continuity across stages. |
| `PG` | Planning and Goal Management | Structures goals, decomposes tasks, and organizes action sequences toward task completion. | Needed when the task requires explicit planning, decomposition, prioritization, or non-trivial coordination. |
| `AE` | Action Execution | Carries out concrete task steps and operational actions in the workflow. | Needed in most tasks that involve producing outputs, transforming data, or executing workflow steps. |
| `IAC` | Interaction and Communication | Manages structured interaction with humans or other systems, including interfaces and communicative exchanges. | Needed when the task depends on human input, conversational exchange, or external communication channels. |
| `LA` | Learning and Adaptation | Incorporates feedback and updates behavior or state based on prior outcomes. | Needed when the task benefits from iterative improvement, feedback integration, or adaptation over time. |
