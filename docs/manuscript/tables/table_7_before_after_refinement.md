# Table 7. Before And After Refinement In The Main Demonstration

Caption: Before and after refinement in the main demonstration.

| Design element | Initial workflow | Refined workflow | Tacit knowledge surfaced |
|---|---|---|---|
| Data interpretation | `Analyze economy` as one coarse step | Separate validation of indicators and assumptions before substantive analysis | Good analysis depends on checking indicator quality and interpretive assumptions, not only running analysis procedures. |
| Visualization | `Create charts` as one coarse step | Separate chart selection, charting rules, and chart drafting | Chart form should follow communicative purpose, comparison structure, and readability constraints. |
| Human oversight | Implicit or absent | Explicit human review gate before final reporting | Review is not merely advisory; it functions as a decision gate for correctness and presentation quality. |
| Revision path | Not explicit | Revision loop from human review back to analysis or chart work | Human critique can trigger structured rework rather than informal ad hoc correction. |
| Publication control | `Publish report` follows directly from writing | Publication rule check inserted before release | Release depends on explicit approval and rule satisfaction rather than automatic completion of drafting. |
| Workflow realism | Linear draft-to-publish pipeline | Multi-step reviewed pipeline with rule-aware checkpoints | Real workflows depend on procedural knowledge that is usually omitted in coarse initial decompositions. |
